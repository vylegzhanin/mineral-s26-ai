# Minimal Vaadin OSS multi-module template

Modules:
- app-kernel: domain/use-case code, mixed Kotlin/Java
- app-contracts: shared DTO and kotlinx.serialization models
- app-edge-ktor: minimal Ktor server
- app-ui-vaadin: Vaadin Flow + Spring Boot + Karibu DSL dependency + file upload demo

Open the repository root `pom.xml` in IntelliJ IDEA.

## Entry points

- Vaadin UI: `com.example.ui.ApplicationKt`
- Ktor server: `com.example.edge.ktor.KtorServerKt`

## Profiles

Runtime profiles for the UI module:
- `dev`: browser launch on, more verbose logs
- `prod`: browser launch off, forward headers enabled, quieter logs

Maven profiles available from the project root:
- `dev`
- `prod`

Examples:

```bash
mvn -Pdev -pl app-ui-vaadin -am spring-boot:run -Dspring-boot.run.profiles=dev
mvn -Pprod -pl app-ui-vaadin -am -DskipTests clean package
mvn -Pprod -pl app-edge-ktor -am -DskipTests clean package
```

## Docker

Files included:
- `docker/ui/Dockerfile`
- `docker/ktor/Dockerfile`
- `compose.yaml`
- `.dockerignore`

Compose profiles:

```bash
docker compose --profile dev up --build
docker compose --profile prod up --build -d
```

## File upload

`MainView` uses the current Vaadin `UploadHandler` API instead of legacy buffer/receiver helpers, so the upload example stays aligned with the non-deprecated upload handling model in modern Vaadin.

Accepted file types:
- `.txt`
- `.json`
- `.csv`
- `.png`
- `.jpg`

Maximum file size: 10 MB.

## Notes

- Maven Wrapper is not included in this archive because it normally needs to be generated by Maven in your environment.
- The UI keeps the Karibu DSL dependency available, but the minimal `MainView` intentionally uses direct Flow components to avoid deprecation warnings around older upload patterns.

Note: collaboration-engine is intentionally allowed for Vaadin 24.5+ because Vaadin states no additional license is required in production from 24.5 onward. Premium components such as Charts/Board/CRUD/RTE remain banned by the enforcer rule.


## Runtime notes
- app-ui-vaadin is packaged as an executable Spring Boot jar via spring-boot-maven-plugin repackage.
- app-edge-ktor includes logback-classic so SLF4J has a provider at runtime.
